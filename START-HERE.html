Part 1 · One-time setup

Get it ready (about 5 minutes)

You only do this once on a computer. If Python is already installed, skip straight to step 2.

1

Make sure you have Python

Open the Terminal app (Mac) or Command Prompt (Windows) and type this, then press Enter:

$ python3 --version

If it prints something like Python 3.11.x, you're set. If not, download it from python.org/downloads — on Windows, tick “Add Python to PATH” during install.

2

Open this folder in the terminal

Type cd followed by a space, then drag this folder onto the terminal window (it fills in the path for you) and press Enter:

$ cd "…/Job Search Tool"
3

Install the pieces it needs

Run this once. It downloads the three small packages the tool relies on:

$ pip install -r requirements.txt
✓

When it finishes without red errors, setup is done — you never have to repeat this on this computer.

Part 2 · Choose what to search

Tell it what you're looking for

Open job_search.py in any text editor (TextEdit, Notepad, VS Code — anything). At the very top is a SETTINGS block. These are the only lines you change:

# The roles you want — add or remove lines freely JOB_TITLES = [ "Sales Representative", "Medical Sales Representative", "Account Executive", ] # Where to search: a city, or "Remote" LOCATION = "San Francisco, CA" # How recent? 24 = last day · 72 = 3 days · 168 = week HOURS_OLD = 72 # Hide titles containing these words (leave [] to keep all) EXCLUDE_TITLE_KEYWORDS = ["Senior", "Director"]
i

Keep the formatting exactly as shown: each title wrapped in "quotes", with a comma at the end of the line. Save the file when you're done. You never need to touch anything below the SETTINGS block.

Part 3 · Run it

Press go and get your list

Back in the terminal (still inside this folder), run:

$ python job_search.py

You'll see it work through each title. After a minute or two it finishes like this:

Searching: Sales Representative ... found 100 Searching: Medical Sales Representative ... found 88 Searching: Account Executive ... found 100 ============================================================ Done! 306 unique job postings saved to: Job_Postings_2026-07-02_1719.xlsx ============================================================
✓

That .xlsx file appears right here in this folder. Double-click to open it. Each run makes a new date-stamped file, so you never lose an earlier search.

Using your results

What every column means

Job Title
The posting's title, exactly as listed on the board.
Company
Who's hiring. Occasionally blank when the board hides it.
Location
City and state, or “Remote.”
Salary
Pay range when the employer published one — otherwise “Not listed.”
Job Type
Full-time, part-time, contract, and so on.
Remote?
Yes or No, when the board flags it.
Date Posted
When it went up. The file is sorted newest-first.
Source
Which board it came from (Indeed, LinkedIn, etc.).
Link
Click to open the posting and apply.
Description
The full job text when the board includes it — otherwise “Click link to find description.”

Three tricks that make it powerful

Sort by salary

Click the Salary filter arrow → Sort. The best-paying roles jump to the top.

Filter to remote

Click the Remote? dropdown and tick “Yes” to see only work-from-home roles.

Focus one company

Use the Company dropdown to isolate every opening at a place you're targeting.

Jump straight to apply

The Link cells are clickable — no copy-pasting URLs.

Good to know

A few honest notes

i

Fewer results than expected? The job boards sometimes throttle automated searches — LinkedIn and Indeed are the most reliable; ZipRecruiter and Glassdoor can come up empty on any given run. Just wait a bit and run it again.

!

“Not listed” salaries are normal. Many employers don't publish pay. Roughly a third of postings include a salary — those show a range; the rest say “Not listed.”

i

Want to search different roles later? Just reopen job_search.py, change the JOB_TITLES or LOCATION, save, and run it again. That's the whole loop.

Under the hood

Powered by JobSpy

The actual searching of the job boards is handled by JobSpy — an excellent open-source library (github.com/speedyapply/JobSpy, MIT-licensed). This tool doesn't replace it; it wraps it. Same proven engine, repackaged so a person — not a programmer — can use it in a minute.

JobSpy on its own gives you

  • A code function with 15+ arguments to call
  • One job title per search
  • A raw 34-column data table
  • Duplicate postings across boards
  • Salary split across four separate columns
  • No spreadsheet — you build and style it yourself

This tool adds on top

  • A plain-English SETTINGS block — no coding
  • Many job titles searched in one run
  • Just the 10 columns that matter, renamed clearly
  • Duplicates removed automatically
  • Clean, readable salaries like “$80,000 – $120,000/yr”
  • A finished Excel: frozen header, filters, clickable links
